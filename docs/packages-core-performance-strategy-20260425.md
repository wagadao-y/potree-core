# packages/core パフォーマンス改善方針 2026-04-25

## 目的

- 2026-04-24 までの CPU 側最適化と、2026-04-25 の Zstd / position+rgb 実測結果を統合する。
- 現在の実装に対して、何が完了済みで、次に何を優先すべきかを 1 本の方針として整理する。
- 旧分析メモに残っている「すでに実装済みの改善」や「優先度が変わった案」を切り分ける。

## 現状認識

- steady-state の FPS は、引き続き GPU time と submitted points に強く支配される。
- 4/24 の CPU 側改善により、可視判定と可視ノード更新の明確な無駄はかなり削れている。
- 4/25 の実測では、Zstd は decompress 単体には効くが、ロード完了後の FPS 改善は限定的だった。
- 一方、属性を position + rgb のみに絞ると、JS heap、attribute decode、transfer bytes は大きく改善した。

したがって、現在の改善方針は次の 2 軸に分ける。

1. steady-state FPS 改善: GPU に送る点数、draw call 数、clip 時の無駄描画を減らす。
2. 初回表示と巨大データ耐性の改善: decode、transfer、heap peak、request 制御を改善する。

## 実装済みとして前提に置く事項

### 可視判定と可視ノード更新の CPU 改善

- `renderer.getSize()` / `getPixelRatio()` の hot loop 内呼び出しは解消済み。
- clip box 判定用の `Box3.clone()` / `new Box3()` / `new Vector3()` は解消済み。
- 前フレームの subtree 全体を隠す方式はやめ、前回の visibleNodes のみを非表示化する方式に変更済み。
- visible nodes texture 更新では、DataTexture 用バッファと child offset バッファを再利用済み。
- `onBeforeRender` の `visibleNodes.indexOf(node)` と name ベースの offset 参照は撤去済み。

### picking 用 indices attribute の通常生成廃止

- picking は `PointColorType.POINT_INDEX` を維持しつつ、shader 側で `gl_VertexID` から point index color を生成する方式に変更済み。
- 通常描画時も picking 時も、loading2 パスでは `indices` attribute を常時生成しない前提でよい。
- この方針は WebGL2 前提である。

### worker payload と metrics の整理

- decoder worker の result message から未使用の raw buffer 返送は削除済み。
- `Raw buffer bytes`、`Generated buffer bytes`、`Precise buffer bytes`、`Transfer buffer bytes` を計測できる。
- `Generated bytes / point` を runtime geometry のメモリ設計指標として使える。

### LOD 制御の現在地

- `screenSpaceDensityLODEnabled` と `maxPointsPerPixel` は既に実装されている。
- そのため、今後は LOD 制御を「新規実装する」段階ではなく、「閾値設計と運用を詰める」段階として扱う。

## 標準データ配信方針

標準ビューア用途の推奨パイプラインは次のとおり。

```text
COPC/LAZ -> Potree v2 dataset
encoding: ZSTD
attributes: position + rgb のみ
```

理由:

- Zstd は decompress と decode throughput を改善できる。
- `position + rgb` 化はファイルサイズ差以上に JS heap、attribute decode、transfer を削減する。
- 不要属性 skip を runtime に持ち込むより、配信データ自体を用途別に分けた方が設計が単純で安定する。
- `intensity`、`classification`、`gps-time` などが必要な用途は、別データセットとして扱う方がよい。

## 重要な設計判断

### 1. FPS 改善の主戦場はロード系ではなく描画系

- Zstd 化や worker payload 整理は、初回表示、decode、heap peak には効く。
- ただし steady-state FPS は `GPU time ~= submitted points` の影響が支配的で、ロード最適化だけでは大きく改善しない。
- したがって FPS 改善の最優先は、GPU に送る点数と draw call 数の削減である。

### 2. position バッファ 12 B/pt の見直しは優先度を下げる

- Int32 直渡し方式は試作済みだが、generated bytes は変わらず、decode と GPU time はわずかに悪化した。
- Int16 量子化は shallow node で精度悪化が大きく、現行データ表現では採用しにくい。
- いま効くのは position の型変更よりも、そもそも送る points 数を減らすことである。

### 3. runtime 側の未使用属性 skip は主戦略にしない

- 圧縮 payload、decompress 後 raw buffer、属性レイアウト解釈の複雑さは残る。
- color mode 切り替え時の再 decode / reload 方針も必要になる。
- 標準配信向けには、データ作成時点で `position + rgb` に絞る方が適切である。

## 現在の優先順位

### 優先度 A: steady-state FPS へ直接効くもの

1. Screen-Space Density LOD の実運用化
   - `screenSpaceDensityLODEnabled` と `maxPointsPerPixel` の既定値、推奨値、UI 露出方針を詰める。
   - まずは submitted points と GPU time の低下量が明確に出る設定帯を見つける。

2. point budget の動的制御
   - dynamic point budget も含め、GPU frame time を見ながら submitted points を抑える。
   - Density LOD と組み合わせて、視点や画面密度に応じて過密な child 展開を止める。

3. draw call 数の削減
   - 現状は draw call 数が CPU render submit と GPU command 処理の両方に効く。
   - 遠距離 node や小サイズ node の統合描画、batch 化の検討を進める。

4. clip-aware な可視制御
   - clip plane / sphere の CPU 粗判定を追加する。
   - clip relation cache と clip-aware point budget を導入し、clip 領域外の点に budget を使わないようにする。

### 優先度 B: ロード中の体感、heap peak、安定性に効くもの

5. Zstd decoder warmup
   - 初回 node の decoder init をロード開始時に先出しし、p95 / max の初回遅延を下げる。

6. bytes in flight 制御
   - `maxNumNodesLoading` だけでなく bytes ベースのバックプレッシャーを導入する。
   - 巨大 node 混在時の heap peak と transfer スパイクを抑える。

7. octree range merge gap 許容
   - 連続 range のみではなく、小さな gap を許容して request 数を削減する。
   - 余分な fetch bytes とのトレードオフは計測で判断する。

8. clip 操作中の uniform 更新コスト削減
   - clip plane / sphere / box の uniform 配列を再利用し、操作中 allocation を減らす。
   - これは clip interaction の入力追従性改善として扱う。

## 優先度を下げる、または現時点で見送る項目

- position を Int32 / Int16 へ変える最適化
- `position + rgb` 専用 fast path の追加
- runtime 側の未使用属性 skip を標準戦略にすること
- visible nodes のソート撤去

これらは効果が限定的、または副作用と複雑さに対して優先度が低い。

- Draw calls
- visible nodes 数
- CPU render submit

- Decompress avg
- Attribute decode avg
- Decode avg
- Transfer avg
- JS heap
- Raw buffer bytes
- Generated buffer bytes
- Precise buffer bytes
- Transfer buffer bytes

- clip 操作中 FPS
- input latency
